      • Commands and Their syntax of GIT

    1. git --version : to check the version of git
    2. Clone : Bring a repositary that is hosted somewhere like github into a folder on your local machine
    3. Add : Track your files and changes in Git.
    4. Commit : Saving the file or changing something in the project
    5. push : Upload Git commits to a remote repo, like GITHUB
    6. pull : Download changes from remote repo to your local machine, the opposite to push

    Configuring Git

    1. git config -- global user.name "Name of the user" (Enter that name you have on Github )
    2. git config --global user.email "email of the user (use that email by which you create your github account)"
    3. git config --list (used to check what we save in ("name and email address"))

    Clone & Status

  • Clone : Clonnig a repositary in our local machine.
  • Command :- git clone ("link of the site or page")
  • Status : Display the status on our local machine that did we changes something code or not.
  • Command :- git status
  • Untracked : New files that git does'nt tracked yet
  • Modified : Changes make in file but not saved/commit
  • Staged : File is ready to be committed
  • Unmodified : Unchanged

  • Adds - Adds new or Changed files in your working directory to the git staging area.

  • commit - it is the record of changes in the file

  • Push - Upload local repo content to remote repo
